const jobRef = push(ref(rtdb, "/jobs"), {

onValue(jobRef, (snap) => {
  const job = snap.val();
  if (job.status === "succeeded") {
    video.src = job.result.signedUrl; // stream from GCS
  } else if (job.status === "failed") {
    toast(job.error.message);
  }
});

How it works

A Firebase-native pattern that’s universal across models.

1

Client writes a job to RTDB

Add a node at /jobs/{jobId} with prompt & params.

2

Extension starts & (if needed) polls

Functions v2 + Firebase Task Queue handle Veo LROs, backoff, and TTL. Sync models resolve immediately.

3

Client listens for results

Subscribe to the node. When status flips to succeeded, you get the result (e.g. GCS signed URL).

Client         ──▶  RTDB /jobs  ──▶  Functions (onCreate)
  │                              │
  │         ◀────────────────────┘  write status=running, opName
  │
  │                              ┌─▶ Task Queue Poller ──▶ Vertex Operations.get()
  │                              │          │
  │                              └──────────┴── (backoff, TTL, retries)
  │
  ◀──────── real-time updates ◀── RTDB /jobs (status/result/error)
